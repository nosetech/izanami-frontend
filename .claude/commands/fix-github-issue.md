# github issueの進め方

1. `gh issue view` で issue 詳細を取得
2. 問題の理解
3. 問題の内容がバグ修正や機能追加であれば、「## バグ修正、機能追加の進め方」にある手順で進めてください。
4. 問題の内容が調査や検討であれば、「## 調査の進め方」にある手順で進めてください。

## バグ修正、機能追加の進め方

1. 関連ファイルの検索
2. 修正の実装
3. テスト実行と検証
4. コミット(developブランチに直接コミットしないこと。feature/\*ブランチにコミットすること。)
5. developブランチへのプルリクエスト作成

## 調査の進め方

1. どのように調査や検討をするか方針を考える。その結果はgithub issuenのコメントに記載する。
2. その方針で良いか承認を求める。
3. 承認されたら、調査や検討を実行する。その結果はgithub issueのコメントに記載する。承認されなかったら、その理由をもとに1の手順をやり直す。
